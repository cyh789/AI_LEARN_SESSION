# 증권 시장 스냅샷 웹앱

## Reference Site Analysis

- `증권 시장 스냅샷 웹앱`의 핵심 가치는 `국내/글로벌 증시 실시간 시세를 빠르게 훑는 경험`에 있다.
- 실제로 이 유형의 앱에서 가장 중요한 것은 화면 디자인보다 `시장 데이터를 거의 실시간으로 지속 수집`하는 능력이다.
- 따라서 이 프로젝트의 1차 핵심은 `코스피`, `코스닥`, `코스피200`, 대표 종목 몇 개를 빠르게 보여 주는 것이다.
- 커뮤니티성 기능은 후순위로 두고, 먼저 `시세 수집`, `갱신`, `표시`, `끊김 복구`를 제대로 설계해야 한다.

## Background

브라우저가 외부 증권 API를 직접 호출하는 방식은 맞지 않는다.

- API 키와 시크릿이 노출된다.
- 사용자 수만큼 외부 API 호출이 중복된다.
- 호출 제한과 재연결 제어를 브라우저에서 안정적으로 처리하기 어렵다.
- 브라우저 환경에서는 공급자별 CORS, Origin, 속도 제한에 직접 노출된다.

그래서 이 앱은 `File-based UI`로 시작하되, 데이터 수집은 반드시 서버 게이트웨이를 통해야 한다.

## Goal

정적 파일로 구성된 가벼운 프론트엔드와, 외부 시장 데이터를 수집하는 얇은 Node 서버를 조합한 시장 스냅샷 웹앱을 만든다.

사용자는 이 앱에서 다음을 할 수 있어야 한다.

- 코스피, 코스닥 등 핵심 시장 지표를 거의 실시간으로 본다.
- 대표 종목의 현재가와 등락률을 같은 화면에서 확인한다.
- 데이터 연결 상태가 정상인지, 지연인지, 끊겼는지 즉시 알 수 있다.
- 관심 자산과 개인 메모를 브라우저에 저장한다.

## Market Data Strategy

### Primary Provider

- 국내 증시 1차 공급자는 `한국투자 Open API(KIS)`로 잡는다.
- 이유:
  - 국내주식 시세와 실시간 WebSocket 계열을 제공한다.
  - 초기 스냅샷용 REST와 장중 갱신용 실시간 채널을 함께 설계하기 좋다.
  - 서버 측 토큰 발급과 접속키 발급 흐름이 분리되어 있어 게이트웨이 구조에 맞는다.

### Optional Secondary Provider

- 크립토가 필요하면 `Upbit WebSocket`을 2차로 붙인다.
- 글로벌 지수는 2차 단계에서 별도 공급자 연동으로 분리한다.

### Not Primary

- `KRX` 공개 데이터는 지연 데이터가 포함될 수 있으므로 `거의 실시간` 핵심 소스로 두지 않는다.

## How It Works

1. 브라우저는 `index.html`, `styles.css`, `app.js` 같은 정적 파일을 로드한다.
2. 브라우저는 외부 증권 API를 직접 호출하지 않고, 우리 서버의 `/api/market/snapshot`만 호출한다.
3. 서버는 시작 시 외부 공급자 인증을 수행하고, 초기 스냅샷 데이터를 수집해 메모리 캐시에 적재한다.
4. 서버는 외부 공급자의 WebSocket 실시간 채널에 연결해 시장 데이터를 지속 수신한다.
5. 서버는 최신 데이터를 내부 공통 포맷으로 정규화한 뒤 브라우저에 `SSE(Server-Sent Events)`로 밀어 준다.
6. 브라우저는 처음 한 번 스냅샷을 렌더링하고, 이후에는 들어오는 업데이트만 카드에 반영한다.
7. 관심 자산, `Long / Short / Watch`, 메모는 브라우저 `localStorage`에 저장한다.

## MVP Scope

### Included

- 코스피, 코스닥, 코스피200 실시간 스냅샷
- 대표 종목 몇 개의 현재가와 등락률 표시
- 서버 측 외부 API 인증 및 재연결
- 서버 메모리 캐시
- `/api/market/snapshot` REST 엔드포인트
- `/api/market/stream` SSE 엔드포인트
- 연결 상태 표시
- 관심 자산 고정
- `Long / Short / Watch` 상태 저장
- 자산별 한 줄 메모 저장
- 수동 새로고침
- 오류 및 지연 상태 안내

### Excluded

- 로그인
- 서버 DB 저장
- 멀티유저 토론 동기화
- 주문/매매 기능
- 푸시 알림
- 글로벌 지수 풀커버리지
- 크립토 실시간 연동 1차 필수화

## First Deliverable

첫 결과물은 `정적 프론트엔드 + Node 데이터 게이트웨이` 조합이다.

- 프론트엔드는 파일 기반으로 단순해야 한다.
- 외부 시장 데이터 호출은 모두 서버에서 수행해야 한다.
- 브라우저는 우리 서버의 스냅샷/스트림 API만 사용해야 한다.
