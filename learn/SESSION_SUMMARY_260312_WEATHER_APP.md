## 5) 현재 위치 및 자동 스케줄러 관련 변경
- `현재 위치 사용` 버튼 클릭 시:
  - 브라우저 `geolocation`으로 현재 좌표 조회
  - 좌표 기준으로 날씨 조회
  - 역지오코딩으로 표시용 지역명 계산
- 자동 스케줄러:
  - 초기에 `5분` 주기였으나 이후 `1분` 주기로 변경
  - ON 시 현재 위치 조회 성공 후에만 활성화
  - 실패 시 자동으로 OFF 처리
- 수동으로 `현재 위치 사용`을 누를 때와 자동 스케줄러 시작 시:
  - 지역명 강제 갱신 로직 추가
- 위치 정확도 개선:
  - `enableHighAccuracy: true`
  - `maximumAge: 0`
  - `timeout: 15000`

## 6) 도시 검색 개선 작업
- Open-Meteo Geocoding API만 사용할 경우 한국 `구` 단위 검색이 잘 되지 않는 문제 확인
- 이를 보완하기 위해:
  - `KOREAN_LOCATION_ALIASES`
  - `REGION_ALIASES`
  를 사용한 검색어 정규화 로직 추가
- 예시:
  - `동작구` -> 내부 검색어 `Seoul`
  - `서울 동작구` -> 내부 검색어 `Seoul`
  - `부산 해운대구` -> 내부 검색어 `Busan`
- 화면 표시 라벨은 원래 검색어를 유지하도록 개선
  - 예: `중구 서울특별시 대한민국`

## 7) 표시 라벨 관련 개선
- 검색 후 선택 지역에 상위 도시명만 남는 문제 수정
- `searchDisplayName` 필드를 추가해 사용자가 입력한 검색어를 별도로 유지
- 검색 결과 라벨은 다음 기준으로 조합
  - 원래 입력값
  - 행정구역(`admin1`)
  - 국가명
- 중복 문자열은 제거하도록 처리

## 12) 현재 제약 및 주의사항
- 이 앱은 브라우저 웹앱이므로 OS 수준 푸시 알림은 제공하지 않음
- 알림은 브라우저 `Notification` API 기반
- 앱이 열려 있을 때만 동작 가능
- 현재 위치 정확도는 브라우저/운영체제 위치 서비스 품질에 영향을 받음
- 데스크톱 환경에서는 GPS가 아니라 Wi-Fi/IP 기반 위치가 잡혀 실제 위치와 오차가 발생할 수 있음
