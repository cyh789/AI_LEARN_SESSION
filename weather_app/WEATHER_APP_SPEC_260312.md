# 날씨 앱 스펙 문서 (2026-03-12)

## 1) 프로젝트 개요
- 앱 이름(가칭): `Morning Weather Brief`
- 구현 위치: `AI_LEARN_SESSION/weather_app`
- 실행 방식:
  - 프론트엔드: `HTML + CSS + JavaScript`
  - 서버: `Node.js` 정적 파일 서버
  - 실행 포트: `8001`
- 목적:
  - 사용자가 아침에 오늘 날씨와 7일 예보를 빠르게 확인할 수 있게 한다.
  - 복잡한 기상 데이터보다 요약 중심의 브리핑 화면을 제공한다.

## 2) 현재 구현 기준
- 현재 앱은 웹 브라우저에서 실행되는 정적 웹앱이다.
- `server.js`가 `localhost:8001`에서 정적 파일을 서빙한다.
- 날씨 데이터는 브라우저에서 외부 날씨 API를 직접 호출해 가져온다.
- 사용자 설정은 브라우저 `localStorage`에 저장한다.

## 3) 핵심 사용자 시나리오
- 사용자는 앱에 접속한다.
- 도시명을 검색하거나 현재 위치를 사용해 지역을 설정한다.
- 앱은 오늘 날씨 요약과 7일 예보를 표시한다.
- 사용자는 아침 브리핑 시간을 저장할 수 있다.
- 사용자는 브라우저 알림 권한을 허용할 수 있다.
- 알림 권한이 허용된 경우, 앱이 열려 있고 브리핑 시간대에 진입하면 브라우저 알림을 보여줄 수 있다.

## 7) 외부 연동
- 지역 검색:
  - Open-Meteo Geocoding API
- 날씨 예보:
  - Open-Meteo Forecast API
- 현재 위치 역지오코딩:
  - Nominatim Reverse Geocoding API

## 8) 알림 동작 정의
- 현재 구현은 브라우저 `Notification` API를 사용한다.
- 사용자가 직접 알림 권한을 허용해야 한다.
- 알림은 브라우저가 열려 있고, 앱이 동작 중일 때만 표시 가능하다.
- 운영체제 푸시 알림이나 백그라운드 예약 발송은 현재 범위에 포함되지 않는다.
- 브리핑 시간은 저장되며, 해당 시간대에 앱이 열려 있으면 알림 발송 조건을 검사한다.

## 9) 자동 스케줄러 동작 정의
- 화면에 `자동 스케줄러` 버튼을 둔다.
- 버튼은 ON/OFF 토글 방식으로 동작한다.
- ON 상태에서는 앱이 열려 있는 동안 `5분마다` 현재 위치 기준 날씨를 다시 조회한다.
- 자동 스케줄러가 동작하면 선택 위치는 현재 위치 기준으로 갱신된다.
- 현재 위치를 사용할 때는 `현재 위치`라는 고정 문구 대신, 역지오코딩 결과의 가까운 도시명을 표시한다.

## 10) 예외 상황 처리
- 도시 검색 결과 없음:
  - 오류 메시지 표시
  - 검색 후보 영역에 재검색 안내 표시
- 위치 권한 거부:
  - 도시 검색 사용 유도
- 날씨 API 실패:
  - fallback 예시 데이터 표시
  - 상태 메시지로 실패 사실 안내
- 알림 권한 거부:
  - 상태 메시지로 안내
- 역지오코딩 실패:
  - 좌표 기반 임시 문자열 표시

## 11) 비기능 요구사항
- 첫 화면에서 주요 정보가 한 번에 보여야 한다.
- 모바일 화면과 데스크톱 화면 모두에서 레이아웃이 깨지지 않아야 한다.
- 외부 API 호출 실패 시에도 화면이 비어 있지 않아야 한다.
- 브라우저 새로고침 후에도 사용자가 저장한 설정이 유지되어야 한다.
- 자동 스케줄러는 중복 interval이 생기지 않도록 한 개만 동작해야 한다.

## 12) 디렉터리 및 파일 구성
- `WEATHER_APP_SPEC_260312.md`: 스펙 문서

- `styles.css`: 화면 스타일
- `app.js`: 날씨 조회, 렌더링, 설정 저장, 알림 처리

## 13) 실행 방법
1. `AI_LEARN_SESSION/weather_app` 디렉터리로 이동한다.
2. `node server.js`를 실행한다.
3. 브라우저에서 `http://localhost:8001/`로 접속한다.

## 14) 현재 구현과 일치하는 성공 기준
- `node server.js` 실행 시 `8001` 포트에서 앱이 열린다.
- 도시 검색 시 후보 목록을 확인하고 선택할 수 있다.
- 도시 검색 또는 현재 위치로 예보를 조회할 수 있다.
- 오늘 요약과 7일 예보가 표시된다.
- 브리핑 시간과 알림 설정이 저장된다.
- 자동 스케줄러를 켜면 5분마다 현재 위치 기준 예보가 다시 조회된다.
- 현재 위치 사용 시 가까운 도시명이 화면에 표시된다.
- 외부 API 호출 실패 시 fallback 데이터가 표시된다.
